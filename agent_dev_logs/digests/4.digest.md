## ЁЯОп Goal
Choose a translation base model тАФ and, more durably, make the translation step
**backend-swappable** so the model choice stays easy to extend, prioritising **free +
low-resource** options (both compute footprint and language coverage). Opening candidates:
GLM-5.2, Cohere Command A, plus the open-source space.

## ЁЯзй Design тАФ pluggable backends behind one seam
A `@runtime_checkable` `TranslationBackend` Protocol (`name`, `supports_glossary`,
`translate(text, src, tgt="en")`, `supported_languages()`) with a `get_backend(name, **kwargs)`
factory. Resolution order: `--backend` тЖТ `config.txt: translation_backend` тЖТ env
`TRANSLATION_BACKEND` тЖТ fallback `"lindat"`. Beyond the Protocol, a backend must also expose a
small **pipeline-compat surface** used by `main.process_single_file`: `vocabulary`,
`reset_protected_count()`, `protected_count`, and (recommended) `license_components(vocab_loaded)`
so paradata licensing stays correct after a swap.

## ЁЯЪЪ Shipped backends
- **`lindat`** (default) тАФ `LindatTranslator`, LINDAT/CUBBITT NMT. `supports_glossary=False` тЖТ
  terminology via the in-process **Tag-and-Protect** + UDPipe; cs-centric (non-`*-en` pairs
  silently rerouted to `cs-en`, `translator.py:435-436`). CC BY-NC-SA.
- **`openai_compatible`** тАФ `LLMTranslator` (Phase 1, the prototype). One OpenAI-compatible REST
  adapter (`/v1/chat/completions`) reaching many free providers via `LLM_BASE_URL` / `LLM_MODEL`
  / `LLM_API_KEY` (OpenRouter free models, Gemini OpenAI-compat, Mistral, Groq, local Ollama).
  `requests`-only (deps in `requirements-llm.txt`). `supports_glossary=True` (prompt-injected тЖТ
  retires Tag-and-Protect for this backend); **OCR-faithfulness guards** (temp 0; strict
  "translate-only, preserve numbers/codes/line structure, do not correct garbled OCR" prompt;
  empty/length-ratio post-check raises `TranslationError` тЖТ skips the file). Knobs at
  `llm_translator.py:80-87`.
- **`ct2`** тАФ `CT2Translator` (Phase 3, scaffold). CTranslate2 int8/int4 self-host for
  **EuroLLM-1.7B/9B** or **MADLAD-400** (both Apache-2.0). **Not registered by default**. Note:
  its heavy deps (`ctranslate2`, `sentencepiece`) are imported **lazily inside methods**
  (`ct2_translator.py:176,193`), so registering the class would only do a light module import тАФ
  the base install stays small without the "commented lines" ritual.

## тЪЦя╕П Licensing recipe (output license computed per run)
Three non-obvious traps make a run NC even when the *model* is permissive:
1. **FastText langid is CC BY-NC 4.0** and **on by default** (`config.txt: source_lang = auto`)
   тЖТ need an explicit `--source_lang` (or a permissive detector).
2. **`amcr_vocab` / `teater_data` are CC BY-NC 4.0** тЖТ need a permissive/empty glossary.
3. The model itself must be permissive.
**Permissive recipe:** `--backend ct2` (Apache-2.0 model) **+** explicit `--source_lang` **+**
permissive/empty vocab. `llm_api` deliberately carries an unrecognised license string ("LLM
provider ToS") тЖТ `para_licenses` treats it as maximally restrictive and warns; the licensing
owner must pin real terms before non-experimental use.

## ЁЯФз Hardening cycle (`802f311`)
- ЁЯза **No silent truncation:** configurable LLM `max_tokens`; CT2 `max_decoding_length` тЖТ 2048.
- ЁЯза **CT2 faithfulness guards:** input/output length-ratio checks (`_MIN/_MAX_LEN_RATIO`).
- ЁЯза **Precision glossary:** Tag-and-Protect matches on strict word boundaries (`\b`).
- ЁЯФМ **API streaming race fixed:** `/translate` buffered into an in-memory `Response` (was
  `FileResponse`-streaming after the `TemporaryDirectory` was deleted); `TestClient` POST coverage added.
- тЪб **XSD compiled once** per run (no per-document network fetch); **CT2 default `int4`** for safe
  consumer-hardware deployment; **dynamic API paradata** (`license_components(vocab_loaded)` not
  hardcoded LINDAT); **Dockerfile** drops test deps + restricts write access to `/cache` + `/data`.
