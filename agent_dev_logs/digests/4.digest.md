## 🎯 Goal
Choose a translation base model — and, more durably, make the translation step
**backend-swappable** so the model choice stays easy to extend, prioritising **free +
low-resource** options (both compute footprint and language coverage). Opening candidates:
GLM-5.2, Cohere Command A, plus the open-source space.

## 🧩 Design — pluggable backends behind one seam
A `@runtime_checkable` `TranslationBackend` Protocol (`name`, `supports_glossary`,
`translate(text, src, tgt="en")`, `supported_languages()`) with a `get_backend(name, **kwargs)`
factory. Resolution order: `--backend` → `config.txt: translation_backend` → env
`TRANSLATION_BACKEND` → fallback `"lindat"`. Beyond the Protocol, a backend must also expose a
small **pipeline-compat surface** used by `main.process_single_file`: `vocabulary`,
`reset_protected_count()`, `protected_count`, and (recommended) `license_components(vocab_loaded)`
so paradata licensing stays correct after a swap.

## 🚚 Shipped backends (verified on `test` HEAD)
- **`lindat`** (default) — `LindatTranslator`, LINDAT/CUBBITT NMT. `supports_glossary=False` →
  terminology via the in-process **Tag-and-Protect** + UDPipe; cs-centric (non-`*-en` pairs
  silently rerouted to `cs-en`). CC BY-NC-SA.
- **`openai_compatible`** — `LLMTranslator` (Phase 1, the prototype). One OpenAI-compatible REST
  adapter (`/v1/chat/completions`) reaching many free providers via `LLM_BASE_URL` / `LLM_MODEL`
  / `LLM_API_KEY` (OpenRouter free models, Gemini OpenAI-compat, Mistral, Groq, local Ollama).
  `requests`-only (deps in `requirements-llm.txt`). `supports_glossary=True` (prompt-injected →
  retires Tag-and-Protect for this backend); **OCR-faithfulness guards** (temp 0; strict
  "translate-only, preserve numbers/codes/line structure, do not correct garbled OCR" prompt;
  empty/length-ratio post-check raises `TranslationError` → skips the file).
- **`ct2`** — `CT2Translator` (Phase 3, scaffold). CTranslate2 int8 self-host for **EuroLLM-1.7B/9B**
  or **MADLAD-400** (both Apache-2.0). **Intentionally not registered by default** (two commented
  lines in `_ensure_registry`) so importing `backend.py` never pulls in `ctranslate2`/`sentencepiece`.

`CTranslate2 int8` is the key low-resource lever (≈4× smaller, 2–8× faster on CPU; supports
NLLB / MADLAD-400 / Opus-MT / M2M-100 / T5). EuroLLM was the notable omission from the first
comment — it covers **all 20** repo languages and is Apache-2.0.

## ⚖️ Licensing recipe (output license computed per run)
Three non-obvious traps make a run NC even when the *model* is permissive:
1. **FastText langid is CC BY-NC 4.0** and **on by default** (`config.txt: source_lang = auto`)
   → need an explicit `--source_lang` (or a permissive detector).
2. **`amcr_vocab` / `teater_data` are CC BY-NC 4.0** → need a permissive/empty glossary.
3. The model itself must be permissive.
**Permissive recipe:** `--backend ct2` (Apache-2.0 model) **+** explicit `--source_lang` **+**
permissive/empty vocab. `llm_api` deliberately carries an unrecognised license string ("LLM
provider ToS") → `para_licenses` treats it as maximally restrictive and warns; the licensing
owner must pin real terms before non-experimental use.

## 🔧 Hardening cycle (current `test` HEAD, `802f311`)
- 🧠 **No silent truncation:** configurable LLM `max_tokens`; CT2 `max_decoding_length` → 2048.
- 🧠 **CT2 faithfulness guards:** input/output length-ratio checks (`_MIN/_MAX_LEN_RATIO`).
- 🧠 **Precision glossary:** Tag-and-Protect matches on strict word boundaries (`\b`).
- 🔌 **API streaming race fixed:** `/translate` buffered into an in-memory `Response` (was
  `FileResponse`-streaming after the `TemporaryDirectory` was deleted); `TestClient` POST coverage added.
- ⚡ **XSD compiled once** per run (no per-document network fetch); **CT2 default `int4`** for safe
  consumer-hardware deployment; **dynamic API paradata** (`license_components(vocab_loaded)` not
  hardcoded LINDAT); **Dockerfile** drops test deps + restricts write access to `/cache` + `/data`.
