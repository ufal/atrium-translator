## Phase B — Live free-LLM smoke (cheap, high signal; key available)
Prove `openai_compatible` on real archival text via **both** entry points.
1. `pip install -r requirements.txt -r requirements-llm.txt`.
2. Export `LLM_BASE_URL` / `LLM_MODEL` / `LLM_API_KEY` (e.g. OpenRouter free model).
3. **CLI:** `python main.py ./data_samples/my_documents --alto --formats alto.xml --backend openai_compatible --source_lang cs --vocabulary data_samples/vocabulary.csv`
   (samples: `MTX201501307_anon.alto.xml` + `C-*.xml`; explicit `--source_lang` avoids FastText langid).
4. **API:** start `service/api.py` (or `TestClient`) with `TRANSLATION_BACKEND=openai_compatible`,
   POST a sample to `/translate`; confirm the in-memory `Response` returns valid XML and `/info`
   records the active backend.
5. **Guards:** temp-0 output, no hallucinated OCR "correction", implausible length-ratio raises
   `TranslationError` (skips file), and the prompt-glossary path fires (`supports_glossary=True`).

**Done when:** a real LLM translation succeeds via CLI **and** `/translate`; default run still `LindatTranslator`.

## Phase C — Run the bake-off (answers the original question)
```bash
python -m eval.bakeoff --backends lindat,openai_compatible \
    --samples data_samples/my_documents --vocabulary data_samples/vocabulary.csv \
    --source_lang cs --limit 40 --out bakeoff.csv
```
- Pass `--source_lang cs` (the default `auto` goes straight into `backend.translate()`; LINDAT is
  cs-centric and silently reroutes unknown pairs to `cs-en`).
- Reference-less QE is the realistic archival case: empty-output rate, **terminology hit-rate**,
  number/code preservation, length-ratio, pairwise char-similarity (CUBBITT↔LLM divergence).
- chrF/BLEU/COMET need a curated `--refs` TSV + `eval/requirements-eval.txt`. **Do not** use the
  existing `data_samples/translated_files/*_en.xml` as references — they are CUBBITT outputs, not gold.
- Spot-check high-divergence rows for hallucination vs. genuine gains.

**Decision to record:** default vs opt-in for `openai_compatible`, written into the doc §6 and the
issue. CUBBITT stays default unless the LLM clearly wins on terminology + faithfulness without
hallucinating OCR.

## Phase D — `ct2` self-host + the permissive-license recipe
1. **Register `ct2` cleanly.** Deps are imported lazily in-method, so register the class
   (unconditionally, or `try/except ImportError`) instead of the "uncomment two lines" ritual —
   self-host users select `--backend ct2` without editing source; base install stays light.
2. **Convert a permissive model** (Apache-2.0): EuroLLM-1.7B (CPU-viable, all 20 langs) first, or
   MADLAD-400-3B. `pip install -r requirements-ct2.txt`, set `CT2_MODEL_DIR` / `CT2_MODEL_FAMILY` /
   `CT2_SP_MODEL` / `CT2_COMPUTE_TYPE`, then translate a sample with `--backend ct2 --source_lang cs`.
   Validate the CT2 guards (`_MIN/_MAX_LEN_RATIO`, `max_decoding_length=2048`).
3. **Permissive recipe end-to-end:** `--backend ct2` (Apache-2.0 model) **+** explicit
   `--source_lang` (no FastText) **+** permissive/empty vocab → confirm
   `para_licenses.resolve_effective_license` computes a permissive output from the logged components.
4. **Flag** that `llm_api`'s placeholder "LLM provider ToS" must be pinned to real terms before
   non-experimental LLM use (licensing owner's call; document the requirement).
